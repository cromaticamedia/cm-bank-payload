## 1. Descripción del Producto

Un Web Site es una presencia digital completa construida sobre un CMS headless (Payload CMS o Strapi), que le permite al cliente gestionar su propio contenido sin tocar código. A diferencia de una landing page estática, este producto escala desde una sola página hasta un sitio multi-ruta con blog, formularios avanzados, múltiples idiomas y optimización de rendimiento.

**Ideal para:** negocios locales, profesionales independientes, startups, consultoras, estudios creativos, restaurantes, clínicas, y cualquier organización que necesite presencia web profesional con autonomía editorial.

## 9. Proceso de Trabajo

Brief & kickoff → Cliente completa formulario + pago del 50%
Arquitectura del sitio → Definición de rutas, bloques y estructura
Wireframe → Presentación de estructura visual (Growth y Scale)
Diseño → Revisión de diseño estático antes de desarrollar
Desarrollo → Implementación con CMS integrado
QA interno → Revisión propia antes de enviar al cliente
Rondas de revisión → Según el tier contratado
Aprobación final → Cliente confirma por escrito
Deploy + transferencia → Publicación y entrega del repo GitHub
Pago del 50% restante → Se libera al publicar
Soporte post-deploy → Según el tier

## 10. Política de Revisiones

- Una **ronda de revisión** = una lista consolidada de cambios enviada en un solo documento o mensaje.
- Cambios de contenido (textos, imágenes) no consumen ronda de revisión.
- Cambios estructurales (nueva ruta, nuevo bloque no contemplado, cambio de layout) se cotizan aparte.
- Cambios solicitados después de la aprobación del wireframe se consideran fuera de alcance y se cobran como trabajo adicional.

## 13. Preguntas Frecuentes del Cliente

**¿Puedo editar el contenido del sitio yo mismo?**  
Sí. Todos los tiers incluyen un Content Dashboard (Payload o Strapi) con el que puedes editar textos, imágenes y secciones sin tocar código, además de videos y documentación para aprender a usarlo.

**¿El repositorio del código es mío?**  
Sí. Al finalizar el proyecto se transfiere el ownership del repositorio GitHub a tu cuenta.

**¿El sitio va a aparecer en Google?**  
El sitio se configura con SEO técnico completo desde Launch (robots.txt, sitemaps, meta tags, JSON-LD). El posicionamiento real depende de factores externos como la antigüedad del dominio y la competencia en tu nicho.

**¿Qué pasa si necesito más rutas o idiomas de los incluidos en mi tier?**  
Están disponibles como add-ons o puedes hacer upgrade al siguiente tier antes de iniciar el proyecto.

**¿Puedo agregar funcionalidades después de entregado el sitio?**  
Sí, se cotiza como un proyecto de extensión según el alcance requerido.

**¿Incluye hosting?**  
Sí, nosotros desplegamos todas nuestras páginas en Cloudflare Pages, es una opción sólida y suficiente para la gran mayoría de sitios web estáticos o con SSR liviano. No es un servicio "de segunda" por ser gratuito, Cloudflare Pages tiene CDN global y TTFB muy bajo por defecto.
